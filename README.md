## Project Overview
This project is a Database Management System designed for a photography equipment rental store.  
The system manages customers, employees, branches, equipment inventory, rental transactions, and billing operations.  
It ensures accurate tracking of rentals, equipment availability, and automatic bill generation based on rental duration and quantity.

---

## System Features
- Manage customer profiles and contact information
- Manage employees and branch managers
- Manage branches and equipment stock per branch
- Record rental transactions with receive and return dates
- Generate bills automatically for each rental
- Calculate total rental cost based on rental duration, quantity, and rental fee
- Generate reports and summaries for rentals and customers

---

## System Users

### Branch Manager
- Add, update, and delete employees
- Add, update, and delete equipment
- View available and out-of-stock equipment
- View most rented equipment
- View customers and spending summaries
- View rental and billing reports
- View active rentals
- View branch information and stock

### Customer
- Create and manage customer accounts
- Browse available equipment
- Search equipment by type
- Create and cancel rentals
- View rental history
- View generated bills
- Run customer-specific queries

---

## Database Design
The system is designed using an Entity Relationship Diagram (ERD) and converted into a relational schema.  
It includes the following main entities:
- CUSTOMER
- EMPLOYEE
- BRANCH
- EQUIPMENT
- RENTAL
- BILL
- STOCK
- MAKES

Primary and foreign keys are used to maintain data integrity and enforce relationships between entities.
